# 03. 调试排坑与经验沉淀 (Troubleshooting History)

[🔙 返回主页](../README.md)

本文件用于记录在项目早期开发与联调过程中遇到的重大“坑点”和算法选型教训。这些经验对后续向更高波特率（如 448G）或更恶劣信道演进时，具有重要的排雷指导意义。

---

## 1. DFE 的雪崩陷阱 (Error Propagation)
在信道极为恶劣（如 Nyquist 处衰减达到 -18dB 且加性噪声较强）时，如果系统的纯线性 FFE 均衡器无法将原始信噪比压制到足够低（例如误码率还在 `1e-1` 级别），此时**绝对不能盲目开启 DFE**。
- **现象**：开启 DFE 后，非但无法降低误码率，反而会导致极其严重的**错误传播 (Error Propagation)**。前一个符号的错误判决会通过反馈抽头持续污染后续数十个符号，导致整个解调链路彻底雪崩。
- **经验原则**：DFE 不是救命稻草。在启用 DFE 之前，必须先将纯线性链路（Tx FFE + Rx FFE）的性能挖掘到极致（物理极限边缘），然后再用 DFE 去清扫残余的 ISI。

## 2. Tx 端模拟滤波器 (IIR) 的相位失真
- **现象**：在发射端 (Tx DSP) 曾短暂尝试引入类似于 CTLE 的 IIR 连续时间滤波器来进行高频提振，结果导致极度严重的非线性相位失真，眼图不仅没有张开反而发生扭曲。
- **经验原则**：LPO 模块的 Host ASIC 发送端通常只使用 FIR 结构的 FFE。FIR 具有线性相位特性，能够干净地实现预加重 (Pre-emphasis) 而不扭曲信号相位。请**严禁在 Tx DSP 中使用任何 IIR 结构的均衡器**。

## 3. Rx FFE 前向抽头对齐 (Precursor Alignment)
- **现象**：曾因为代码逻辑漏掉了前向抽头的延时补偿，导致 Rx FFE 的主光标 (Main Cursor) 对齐在了第一个 Tap 上，使得 FFE 完全丧失了消除前向 ISI (Pre-cursor ISI) 的能力，性能断崖式下跌。
- **经验原则**：在构建分数间隔 (Fractional-spaced) FFE 时，必须严格管理 `ffe_pre` 参数并进行正确的历史状态延时，确保主光标能够落在数组的中心偏后位置。

## 4. DFE 与 MLSE 的相互“背刺”
- **现象**：在使用 Burg AR 拟合残余色噪并送入 MLSE 进行联合译码时，发现 MLSE 相比于 Slicer 毫无性能提升（编码增益近乎为 0）。
- **根本原因**：系统同时开启了 1-tap DFE 和 1-tap MLSE。DFE 会通过硬判决直接吃掉最核心的“第一后沿 ISI”，导致残余误差序列变得极其接近白噪声。Burg 算法在这种情况下只能拟合出极弱的相关系数，MLSE 丧失了用来解码的强 ISI 支撑。
- **经验原则**：MLSE 必须与完全线性的迫零 (ZF/MMSE) FFE 配合。绝对禁止在开启 MLSE 的相关抽头长度内混用 DFE。在目前的实现中，一旦检测到开启 MLSE，系统将自动防呆锁死 DFE。

## 5. 贝叶斯优化器 (BO) 的过拟合死锁 (GP Over-Exploitation)
- **现象**：在高维空间 (如 10D 的 Tx FFE 优化) 中，纯依赖 Expected Improvement (EI) 的 BO 会锁死在默认的初始点，连续十几次搜索均返回该点附近。
- **根本原因**：边缘似然 (LML) 为了最大化似然值，将高斯过程 (GP) 的长度尺度优化得过大，导致模型认为空间中只有默认点这一根针，其他区域异常平坦，EI 梯度归零。
- **经验原则**：严禁把黑盒求解器用于物理空间的参数寻优。必须采用白盒化 BO，对长度尺度 (Length Scale) 进行硬性物理边界限制，并将采集函数从保守的 EI 替换为 LCB (Lower Confidence Bound) 进行强制定向爬坡，方能打破高维搜索死锁。
