# 04. 优化算法架构与原理解析

[🔙 返回主页](../README.md)

本项目在发送端 (Tx FFE) 权重寻优中，彻底摒弃了第三方优化库的黑盒调用，完全使用 Numpy/Scipy 从零构建了四套各具特色的优化算法。针对不同的测试场景和在线/离线要求，可以在 `config.xlsx` 的 `[tx]` 表格中通过 `optimizer_type` 无缝切换。

---

## 1. 算法矩阵一览

| 缩写 | 全称 | `optimizer_type` | 核心特性 | 适用场景 |
| :--- | :--- | :--- | :--- | :--- |
| **BO** | 贝叶斯优化 (Bayesian Optimization) | `'BO'` | 建立全局高斯过程模型，通过期望增益 (EI) 最大化来平衡全局探索和局部开发。 | **离线全局探索**：对信道完全未知，希望用最少的计算次数探明全局最优解。 |
| **GA** | 连续型遗传算法 (Genetic Algorithm) | `'GA'` | 基于种群演化，引入锦标赛选择、交叉重组和多点高斯变异。 | **大范围离线演化**：算力充足，希望通过多点并行的变异彻底跳出局部最优陷阱。 |
| **SA** | 受限模拟退火 (Bounded Simulated Annealing) | `'SA'` | 单点局部随机漫步，引入退火概率和硬性退化阈值。 | **中等范围搜索**：起点一般，允许容忍一定程度的性能退化以穿越平坦区。 |
| **SHC** | 安全微步爬山 (Safe Micro-Step Hill Climbing) | `'SHC'` | 极小步长限制的局部梯度探索，实时动态缩减步长。 | **在线实时调优**：硬件链路在线跑业务时，需要无感微调，**绝对禁止**尝试任何导致链路断开的恶化参数。 |

## 2. 核心算法原理解析

### 2.1 安全微步爬山算法 (SHC - Safe Hill Climbing)
在线实装链路调优的杀手锏。不同于离线仿真可以肆无忌惮地输入极差参数测试（会导致高达 30% 误码率），SHC 采用了两重安全保障：
1. **跳过随机初始化**：取消全局随机探雷（Random Init），直接基于当前已知的最优起点（如 `[0, 0, 0, -0.2987, 0.7012, 0, 0, 0, 0]`）起步。
2. **退化阈值动态折半**：限制探索步长极为微小（`initial_step_size = 0.01`）。一旦发现试探点带来的 FFE 误码率超过了历史最佳的 5 倍（`max_regression_ratio = 5.0`），不仅直接拒绝该参数，还会立刻将试探步长再缩小一半，死死把性能波动压制在安全范围内。

### 2.2 连续型遗传算法 (GA)
对于连续的浮点型滤波器权重（且存在 $\sum |w| = 1$ 的 L1-Norm 约束），传统的离散遗传算法难以适用。我们采用了连续空间实数编码机制：
- **锦标赛选择 (Tournament Selection)**：随机抽取 3 个个体，让适应度最好的胜出成为父母，保证基因优良的同时维持种群多样性。
- **算术交叉 (Arithmetic Crossover)**：产生插值子代 $O = \alpha \cdot P_1 + (1-\alpha) \cdot P_2$。
- **动态重投影**：每次突变后，通过 L1-Norm 归一化强行将基因序列拉回合法的发送端能量约束球面上。

### 2.3 受限模拟退火 (SA)
在标准模拟退火的基础上，为了满足“不要回退太多”的物理场景需求，我们加入了**双重校验门**：
哪怕满足了 $e^{-\Delta E / T} > \text{rand()}$ 的概率接受条件，新解的绝对恶化程度也必须满足 $\text{BER}_{\text{new}} \le 5.0 \times \text{BER}_{\text{best}}$。这避免了高温早期退火算法跑到完全不可用的死角。

### 2.4 贝叶斯优化 (BO)
这是最复杂的白盒数学实现：
- **ARD RBF 协方差核 (Kernel)**：自行推导了支持各向异性（Anisotropic）特征长度的径向基核函数。
- **超参数内环 Adam 优化**：不依赖 scipy 的 L-BFGS-B，而是自己手写了 Adam 梯度下降，针对负对数边际似然函数 (Negative Log Marginal Likelihood) 的有限差分梯度，实时优化高斯过程的 $\sigma_f$ 和长度尺度 $l$。
- **GS-EI 获取函数优化**：采用了两阶段寻找下一个测试点：全局多重起点随机撒网粗筛，结合针对 EI（Expected Improvement）函数的数值投影梯度上升进行精筛。

## 3. 目标函数 (Fitness Function) 的连续化改造
在早期版本中，我们发现由于 MLSE (最大似然序列估计) 的硬判决特性，在 50000 符号的仿真长度下，局部极小值附近（如 9.88e-04，精确的 49 个错误）会形成一片**梯度绝对为 0 的绝对平原**。

为解决此问题，目前所有优化算法提取的 `y_data` 目标函数已经**全面替换为送入 MLSE 之前的 FFE 软判决误码率**：
```python
# target_obj = math.log10(max(ffe_ber, 1e-8))
```
这为算法提供了极其平滑、连续的下降指引（Gradient），而在日志打印和图表输出时，依然会如实汇报最终的 MLSE 硬判决系统总误码率。

## 4. 高保真性能大考 (200万符号打擂台)

为了严格验证不同算法在线或离线寻优的安全性与极限性能，我们提升了环境保真度：将 `snr_db` 设为 27.5 dB，同时将单次评估符号数 `num_symbols` 暴增至 200 万 (2,000,000)，从而让 1e-6 级别的底层极低误码率得以在仿真中暴露。

四种算法同时在初始最优起点的基础上，各迭代 40 次进行极限摸底寻优。其实测收敛性能与试探点恶化上限（Max FFE BER 测试）的对比如下：

| 优化算法 | 最优 MLSE BER | 最优 FFE BER | 寻优期间试探出的最差参数 (Max FFE BER) | 安全性评级 |
| :--- | :--- | :--- | :--- | :--- |
| **BO (贝叶斯)** | `2.01e-05` | `7.30e-03` | `3.41e-01` **(高达 34.1% 误码率)** | ❌ 灾难掉线 (全局乱跳) |
| **GA (遗传)** | `2.01e-05` | `7.30e-03` | `3.13e-01` **(高达 31.3% 误码率)** | ❌ 严重断流 (大步长变异) |
| **SA (退火)** | `2.01e-05` | `7.30e-03` | `2.11e-02` (退化约 2.9 倍) | ⚠️ 勉强可用 (有局部回退) |
| **SHC (微步爬山)** | `2.01e-05` | `7.30e-03` | `1.00e-02` **(仅退化 1.3 倍)** | ✅ **工业级在线安全无感调参** |

**结论**：在当前 112G 纯线性约束下，最优解 `[-0.2987, 0.7012]` 确实已是物理全局最优极限。四种算法都能找到该点，但对于**在线硬件链路**的寻优而言，只有 `SHC` 算法死死将参数的中间试探恶化控制在了安全阈值（<1.5倍）内，彻底杜绝了盲目探索导致的光纤断流，完美满足了实时在线调参的需求。
